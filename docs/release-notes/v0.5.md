# v0.5 Release Notes

## Summary

Version 0.5 promotes internal PKI and ingress changes that were validated
operationally during rebuild and cutover testing.

## Highlights

- step-ca is now the active internal ACME issuer backend via:

      ClusterIssuer/step-ca-int-acme

- ACME directory endpoint standardized to:

      https://step-ca.step-ca.svc.cluster.local/acme/acme/directory

- `deploy-all.sh` supports phased execution:
  - `DEPLOY_PHASE=pulumi`
  - WireGuard setup/validation pause
  - `DEPLOY_PHASE=platform`

- WireGuard gateway enhancements:
  - Optional private ENI attachment from Pulumi
  - Multi-interface masquerade support in gateway setup script
  - Policy-routing priority fix (`lookup main` at priority 100)

- ingress-nginx private controller moved to internal AWS NLB
  (`Service` type `LoadBalancer`) with internal annotations.

- Argo CD internal ingress now requests cert-manager certificates from
  `step-ca-int-acme` and serves HTTPS through ingress.

- Kustomize validation script updated for CI resiliency:
  - retries for transient helm/chart fetch failures
  - validation fallback for encrypted step-ca secret when SOPS age key is not present

## Notes

- Existing manual drift in issuer account-key secret names can still be recovered
  by deleting stale `step-ca-int-acme-account-key*` secrets and reconciling
  the ClusterIssuer.
- Internal DNS and WireGuard pathing remain mandatory for private ingress access.
